- MUST ensure backend-check and frontend-check are passing before committing
- Interface Segregation / Least Privilege: functions and types must accept only what they actually need. Never pass a large struct (e.g. `AuthContext`) when a single field (e.g. `org_id: Uuid`) is all that's used.
- MUST follow SOLID and DRY principle.
- MUST follow atomic design: always compose UI from existing design system components (`frontend/src/components/ui/`). Never write raw HTML elements (`<button>`, `<input>`, `<select>`, `<textarea>`, `<div>` for layout primitives) when a component already exists — use `Button`, `InputGroup`, `SelectGroup`, `TextareaGroup`, `Card`/`CardHeader`/`CardContent`/`CardFooter`, `Progress`, `RadioGroup`/`RadioItem`, etc. Build new UI bottom-up: atom → molecule → organism → page. If a needed component does not exist, create it in `components/ui/` before using it in a feature.
- MUST emit structured, metadata-rich tracing logs. Every log call must include all relevant IDs and context as structured fields — never interpolate them into the message string. Backend: `tracing::info!(session_id = %id, org_id = %org_id, stage, story_id = ?story_id, "description")`. Frontend: `console.error('[component/hook]', { field1, field2 }, error)`. At minimum include: entity IDs (story_id, task_id, session_id, org_id, project_id), stage/state, and the operation being performed. Use `%` for Display types and `?` for Debug types. Log at entry and exit of every significant async operation (dispatch, DB write, WS send) plus all error branches.
- MUST NOT silently swallow errors. Every error path must be observable: use `tracing::error!` / `tracing::warn!` on the backend; surface errors to the user or log them via `console.error` on the frontend. Patterns like `.unwrap_or_default()`, `unwrap_or("")`, `unwrap_or(vec![])`, and empty `Err(_) => {}` arms are only acceptable when the fallback is intentional and documented with a comment. Missing SSE event handlers, unhandled match arms, and fire-and-forget async calls that drop errors are bugs, not style choices.
